Chapter 3 Using this Guide

3.1 The Basics

The functions in eyeTrackR are mostly split up into two categories: Organise functions and Analyse functions.1

Organise functions are there to help you clean, check, and, well, organise your data.

Analyse functions are there to work out means of common dependent variables and put them together in a format that can easily then be analysed by standard statistical methods (e.g., ANOVAs), as well as visualising those dependent variables.

For most of my projects, I have two key sets of files:

  1. **ORGANISE__markup.R** - this marks up the dataset and cleans it, as you’ll see in this chapter.

  2. **ANALYSE__XYZ.R** - this either contains all of my analyses (if there aren’t that many), OR I split this file up, so that I have one **ANALYSE__XYZ.R** file for each dependent variable.

In this chapter, and the next, I’ll walk you through the steps for putting together a set of analyses for a project.

3.2 Getting Started

To start with, you’ll need to output three different types of report from SR Research DataViewer. You’ll need:

  • Fixation report
  • Message report
  • Interest area report

3.3 Installing via Devtools

From R using the devtools package:

library(devtools)
install_github('hjgodwin/eyetrackR/source')
